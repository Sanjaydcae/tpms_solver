# PrePoMax Reference Plan for TPMS Studio

TPMS Studio should use PrePoMax as an architectural reference, not as copied source code.
PrePoMax is GPL-3.0, while TPMS Studio is currently marked as proprietary software of
H2one Cleantech Private Limited. Copying GPL code into this repository would create a
license conflict unless the whole product is relicensed compatibly.

## What To Adopt Conceptually

- Separate the application into CAE layers:
  - `CaeMesh` equivalent: nodes, elements, element sets, node sets, mesh quality.
  - `CaeModel` equivalent: materials, sections, steps, loads, boundary conditions.
  - `CaeResults` equivalent: field outputs, result components, result metadata.
  - `CaeJob` equivalent: solver job creation, progress, logs, abort/re-run.
  - `vtkControl` equivalent: VTK viewport, scalar bars, camera, clipping, probes.
- Treat the GUI as a controller over model commands, not as the owner of simulation data.
- Prefer external-quality meshing and solving backends over handmade voxel approximations.

## Immediate Corrections Needed

1. Mesh view must show actual Tet4/C3D4 connectivity, not only the exterior surface.
2. Volume meshing must move from voxel-split tetrahedra to surface-conforming tetra meshing.
3. Solver output must be marked as preliminary until benchmarked against CalculiX/PrePoMax.
4. Result plots must report which solver and element formulation produced the data.
5. Stress plots must include a warning when constrained-node stress peaks dominate the range.

## Meshing Direction

Current implementation:

- Surface: generated as triangles from an implicit TPMS field.
- Volume: generated by splitting occupied voxels into Tet4 elements.

Problem:

- The cells are Tet4 internally, but the mesh is not a professional surface-conforming tet mesh.
- It creates staircase/blocky geometry and poor stress quality.

Target implementation:

- Keep only:
  - Surface mesh: `Tri3`
  - Volume mesh: `Tet4 / C3D4`
- Add a real tetra backend:
  - Preferred first option: Gmsh CLI integration.
  - Preferred second option: NetGen integration.
- Workflow:
  - Generate TPMS surface.
  - Export watertight STL/OBJ.
  - Call external mesher.
  - Import nodes and C3D4 tetrahedra.
  - Run mesh quality checks.

## Solver Direction

Current implementation:

- Internal linear-static solver.
- Penalty constraints.
- Tet4 stiffness assembly.
- Basic stress recovery.

Problem:

- Good for workflow demonstration.
- Not yet validated enough for engineering decisions.

Target implementation:

- Add a CalculiX-compatible job path:
  - Export `.inp` with C3D4 elements.
  - Run `ccx` if installed.
  - Parse `.frd`/`.dat` outputs.
  - Display displacement, stress, strain, and reaction forces.
- Keep internal solver only as a fallback/demo solver.

## Visual Direction

- Use VTK for the main viewport.
- Mesh mode should show internal tetra wireframe.
- Result mode should show smooth boundary contours with optional mesh edge overlay.
- Add PrePoMax-style controls:
  - deformation scale,
  - scalar range lock,
  - min/max markers,
  - clipping plane,
  - probe,
  - result component selector.

## Validation Direction

Before claiming the solver is correct, add benchmark cases:

- Single Tet4 patch test.
- Cantilever beam displacement benchmark.
- Uniaxial compression block benchmark.
- 15 mm lattice compression compared against CalculiX.
- Reaction force balance check: applied force plus support reaction should be near zero.
